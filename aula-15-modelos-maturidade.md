# Aula 15 - Modelos de Maturidade

## 1. Diagnóstico de Maturidade

| Critério                                                          | Avaliação |
| ----------------------------------------------------------------- | --------- |
| Os requisitos são documentados?                                   | Parcial   |
| Existe controle de mudanças?                                      | Não       |
| Há atividades de teste definidas?                                 | Sim       |
| Os defeitos são registrados?                                      | Parcial   |
| O processo de desenvolvimento é conhecido por toda a equipe?      | Parcial   |
| As tarefas são planejadas e acompanhadas regularmente?            | Parcial   |
| Existe padronização para implementação de funcionalidades?        | Não       |
| Os testes são executados antes da entrega das funcionalidades?    | Sim       |
| Há revisão de código ou validação por outro integrante da equipe? | Parcial   |
| A equipe utiliza ferramentas para gerenciamento das atividades?   | Sim       |
| Os artefatos do projeto são organizados e versionados?            | Sim       |
| Existe rastreabilidade entre requisitos e funcionalidades?        | Não       |
| A equipe realiza retrospectivas para identificar melhorias?       | Parcial   |
| Existem métricas para acompanhar a qualidade do projeto?          | Não       |

### 📊 Classificação do Processo

**Nível: Gerenciado**

### 📝 Justificativa

O processo apresenta algumas práticas organizadas, como uso de ferramentas, versionamento e execução de testes. No entanto, ainda existem lacunas importantes, como ausência de métricas, falta de padronização e rastreabilidade. O processo não é totalmente definido nem seguido de forma consistente por todos, caracterizando um nível intermediário de maturidade.

---

## 2. Lacunas Identificadas

| Lacuna                                | Impacto                                   |
| ------------------------------------- | ----------------------------------------- |
| Falta de métricas de qualidade        | Dificulta análise e melhoria contínua     |
| Ausência de padronização de processos | Gera inconsistência e retrabalho          |
| Falta de rastreabilidade              | Complica manutenção e evolução do sistema |
| Controle de mudanças inexistente      | Risco de erros e perda de informações     |

---

## 3. Propostas de Melhoria

| Melhoria                                        | Benefício                                     |
| ----------------------------------------------- | --------------------------------------------- |
| Definir e documentar o processo padrão          | Maior organização e alinhamento da equipe     |
| Implementar métricas de qualidade               | Melhor controle e tomada de decisão           |
| Criar rastreabilidade entre requisitos e código | Facilita manutenção e evolução                |
| Adotar code review obrigatório                  | Redução de erros e aumento da qualidade       |
| Implementar CI/CD                               | Automação e maior confiabilidade nas entregas |
